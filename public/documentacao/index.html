Guia de implantacao e configuracao do sistema

Este material cobre a ordem recomendada de implantacao do sistema, os ajustes obrigatorios no servidor, as configuracoes internas do painel, os comandos cron e a leitura pratica das integracoes existentes.

Laravel 12 Base principal do sistema, compatível com PHP 8.4+.
Filament 4 Painel administrativo multiunidade com rotinas operacionais.
Sem Node obrigatorio O pacote entregue roda sem build frontend adicional no servidor.

1. Requisitos do servidor

  • PHP 8.4 ou superior.
  • MariaDB recomendado. SQLite funciona para teste rapido.
  • Extensoes PHP: pdo, pdo_mysql, openssl, mbstring, json, fileinfo, curl, gd, zip.
  • Permissao de escrita em storage/ e bootstrap/cache/.
  • Suporte a cron do cPanel.
  • HTTPS ativo para portal, webhooks e PWA.
Recomendacao pratica: use PHP 8.4 com extensoes completas, banco MariaDB separado por ambiente e certificado SSL ativo antes de abrir o portal do paciente.

2. Implantacao no cPanel / CloudLinux

  1. Envie o arquivo ZIP do projeto e extraia dentro da pasta da aplicacao.
  2. Garanta que o arquivo index.php e o .htaccess da raiz permaneçam ativos para ocultar /public da URL.
  3. Crie o banco de dados MariaDB no cPanel e registre nome, usuario, senha e host.
  4. Defina a versao do PHP da conta para 8.4 ou superior.
  5. Garanta permissao de escrita para storage e bootstrap/cache.
  6. Se usar subdominio, aponte o document root para a raiz do projeto, nao para /public.
composer install --no-dev --optimize-autoloader
cp .env.example .env
# edite o .env com os dados reais do MariaDB
php artisan key:generate
php artisan storage:link
php artisan migrate --seed
Se o arquivo .env nao existir, o Laravel vai cair nos fallbacks do projeto e pode tentar usar SQLite. Em producao, crie o .env primeiro e configure o MariaDB antes de rodar migrate.

3. Instalador web

Acesse /instalar. O instalador valida requisitos, grava o .env, testa banco, executa migracoes, seed e cria o primeiro superadmin.

Etapa O que preencher Observacao
Aplicacao Nome do sistema, URL base, timezone, ambiente. Use a URL final em producao com HTTPS.
Banco Host, porta, nome do banco, usuario e senha. MariaDB em producao. SQLite apenas para homologacao.
Superadmin Nome, e-mail e senha do primeiro acesso. Use um e-mail real para futuras rotinas administrativas.

4. Pos-instalacao e ordem de configuracao no painel

  1. Configuracoes do sistema: nome da aplicacao, rodape, versao, tour inicial, regras globais do WhatsApp.
  2. Unidades: cadastro completo, CEP, telefone, WhatsApp, dados fiscais, parametros de NFSe.
  3. Usuarios e perfis: superadmin, admin-unidade, recepcao, dentista, financeiro e estoque.
  4. Profissionais: vincule o usuario, CRO, especialidade, cor da agenda e percentual de comissao.
  5. Cadeiras / salas: organize os recursos que entram no conflito de agenda.
  6. Procedimentos e planos: base de atendimento, duracao, preco e regras de consumo.
  7. Convenios: configure ANS, documento, canal padrao, validade de autorizacao e se o plano exige guia previa.
  8. Templates de notificacao: WhatsApp operacional, lembretes, cobrancas, reativacao e assinatura.
  9. PWA: nome do app, icones, push e comportamento do portal quando instalado.
  10. Financeiro: Mercado Pago, parcelamentos internos, repasses e BI.
  11. Faturamento fiscal: valide dados fiscais da unidade antes de emitir NFSe.

6. Governanca clinica

O sistema possui uma central dedicada a qualidade assistencial, pensada para evitar atendimento concluido sem fechamento clinico e tratamento aprovado sem continuidade operacional.

  • Consultas concluidas sem prontuario clinico.
  • Pacientes com tratamento aprovado e sem retorno futuro agendado.
  • Itens do plano vencidos e ainda nao concluidos.
  • Pacientes em tratamento com documentacao obrigatoria pendente.

7. Convenios, guias e retorno operacional

O sistema agora possui uma central propria para autorizacoes de convenio, conectada aos planos de tratamento e aos procedimentos que exigem aprovacao previa.

  • Leitura de planos aprovados elegiveis para montar guia.
  • Critico automatico por convenio que exige autorizacao ou por procedimento marcado com aprovacao obrigatoria.
  • Fila com rascunho, envio para operadora, retorno autorizado, parcial, negado e expirado.
  • Exportacao JSON estruturada pronta para futura integracao TISS.
  • Expiracao automatica das guias vencidas para evitar execucao clinica fora da validade.
Fluxo recomendado: configurar o convenio, aprovar o plano de tratamento, gerar a guia, enviar para a operadora, registrar o retorno e so depois executar o item clinico autorizado.

8. Faturamento de convenio e lote TISS-ready

Depois da etapa de autorizacao e execucao clinica, o sistema possui uma central especifica para transformar itens autorizados em lotes de faturamento por convenio e competencia.

  • Agrupamento automatico por unidade, convenio e competencia mensal.
  • Criacao de lote em rascunho com guias internas e itens faturados.
  • Envio do lote com numero interno e trilha de submissao.
  • Registro de retorno da operadora com valor recebido, glosa parcial ou glosa total por item.
  • Reapresentacao de glosa em novo lote, mantendo referencia ao item original.
  • Exportacao JSON estruturada em formato interno TISS-ready, pronta para futura integracao real com operadoras.
php artisan clinic:insurance-claims-create-drafts --competence=2026-03
Fluxo recomendado: executar o atendimento, gerar os lotes em rascunho da competencia, revisar o agrupamento, enviar para operadora, registrar o retorno e reapresentar glosas quando necessario.

9. Central LGPD e solicitacoes do titular

O sistema possui uma central administrativa dedicada a privacidade, com exportacao estruturada do cadastro do paciente e anonimizacao assistida do registro principal.

  • Tipos de solicitacao atuais: exportacao de dados e anonimizacao do cadastro.
  • Download do pacote JSON em rota protegida do admin.
  • Anonimizacao cobre cadastro do paciente, responsaveis, usuario do portal e assinaturas PWA.
  • Registros clinicos, financeiros, documentais e fiscais permanecem preservados sob base legal de retencao.
Fluxo recomendado: registrar a solicitacao na Central LGPD, processar internamente, baixar o pacote JSON quando for exportacao e registrar observacoes do atendimento ao titular.

10. Cron e scheduler

O modelo recomendado em hospedagem compartilhada e usar um unico cron chamando schedule:run a cada minuto. O proprio Laravel dispara as rotinas internas nas frequencias corretas.

* * * * * /usr/local/bin/php /home/USUARIO/public_html/Gestor_Odontologico/artisan schedule:run >> /dev/null 2>&1
Comando Frequencia recomendada Finalidade Necessario?
php artisan schedule:run A cada 1 minuto Aciona o scheduler do Laravel, incluindo automacao operacional e fila fiscal. Obrigatorio
php artisan clinic:automation-run A cada 15 minutos Roda lembretes, cobranca preventiva e reativacao. Opcional direto. Ja esta dentro do scheduler.
php artisan clinic:nfse-submit --limit=30 A cada 15 minutos Processa a fila de NFSe pendente e gera protocolo. Opcional direto. Ja esta dentro do scheduler.
php artisan clinic:insurance-authorizations-expire A cada 1 hora Expira automaticamente guias de convenio que passaram da validade. Opcional direto. Ja esta dentro do scheduler.
php artisan clinic:insurance-claims-create-drafts --competence=AAAA-MM 1 vez por madrugada ou por competencia em rotina controlada Gera lotes em rascunho do faturamento de convenio para itens autorizados e executados. Opcional direto. Ja esta dentro do scheduler.
php artisan clinic:commission-sync Manual ou 1 vez por madrugada Reprocessa comissoes de contas pagas antigas. Opcional.
php artisan clinic:commission-close --from=AAAA-MM-01 --to=AAAA-MM-31 Manual ou cron mensal customizado Fecha lotes de repasse por periodo informado. Opcional e exige periodo explicito. 
*/15 * * * * /usr/local/bin/php /home/USUARIO/public_html/Gestor_Odontologico/artisan clinic:automation-run >> /home/USUARIO/logs/automation.log 2>&1
*/15 * * * * /usr/local/bin/php /home/USUARIO/public_html/Gestor_Odontologico/artisan clinic:nfse-submit --limit=30 >> /home/USUARIO/logs/nfse.log 2>&1
15 * * * * /usr/local/bin/php /home/USUARIO/public_html/Gestor_Odontologico/artisan clinic:insurance-authorizations-expire >> /home/USUARIO/logs/convenios-expire.log 2>&1
30 1 * * * /usr/local/bin/php /home/USUARIO/public_html/Gestor_Odontologico/artisan clinic:insurance-claims-create-drafts --competence=$(date +\%Y-\%m) >> /home/USUARIO/logs/convenios-faturamento.log 2>&1
5 2 * * * /usr/local/bin/php /home/USUARIO/public_html/Gestor_Odontologico/artisan clinic:commission-sync >> /home/USUARIO/logs/commission-sync.log 2>&1
Se voce usar o cron principal com schedule:run, nao precisa repetir os comandos clinic:automation-run, clinic:nfse-submit, clinic:insurance-authorizations-expire e clinic:insurance-claims-create-drafts em crons separados.
O comando clinic:commission-close nao deve ser automatizado sem uma politica clara de periodo de fechamento. O ideal e executar manualmente no fechamento financeiro de cada unidade.

11. Preciso rodar Node.js?

Para colocar este projeto em producao do jeito que foi entregue, nao. O sistema atual nao depende de build frontend em tempo de execucao para funcionar no servidor.

  • Existe package.json no projeto porque a base Laravel traz Vite para desenvolvimento.
  • Nesta entrega, o sistema nao usa @vite nas views principais.
  • Voce so precisa de Node.js se for alterar frontend, recompilar assets ou iniciar ambiente local com Vite.
npm install
npm run dev
# ou, para build local
npm run build
Resposta curta: em producao, com o pacote entregue, voce nao precisa rodar nenhum comando de Node.js para o sistema subir.

12. Boas praticas finais antes de abrir o sistema

  • Trocar a senha inicial do superadmin.
  • Configurar SSL e forcar acesso HTTPS.
  • Validar rodape, nome do desenvolvedor e versao do sistema.
  • Revisar regras de horario de acesso por perfil.
  • Validar opt-in de WhatsApp antes de disparos operacionais.
  • Conferir unidade, dados fiscais e templates antes de emitir NFSe.
  • Testar o cron principal e acompanhar logs nas primeiras 24 horas.
